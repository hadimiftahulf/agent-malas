# Development Guidelines for Agent Malas

- Write clean, maintainable code.
- Ensure any changes do not break existing functionality.
- Prioritize security: do not commit any secrets, hardcoded credentials or API keys.
- Write code in a style consistent with the current repository.
- Include proper error handling for all asynchronous operations.
- Provide descriptive commit messages.

## Additional Context

You are acting as an autonomous developer agent. Analyze the codebase deeply before making changes.
